### Proposals and discussions

####  1. Proposals licensing (5 minutes, https://github.com/WebAssembly/spec/pull/2050)

RH:  We don’t have licenses for markdown files created by people who make proposals for new features, should be rectified. CC0 is Mozilla’s preference.

TL: this would only apply to proposals once they’re merged into the spec repo? Or do we want to cover all proposals that get created?

AR: since this lives in all the forks, once the changes downstream to the fork repos, it applies there.

CW: We have a decent amount of leeway because we require people to be CG members when they merge, so we can pick a license that the forks inherit.

TL: So they would eventually propagate.

CW: Let’s ask for unanimous consent to make the proposal text in the markdown summary files to be CC0.

No objections. Please reach out to the chairs if you want to follow up.

####  2. Adding Ryan Hunt as a WebAssembly JS-API editor (5 minutes)

RH: ms2ger has been our spec editor, I think we should have another editor for the spec to spread the load out more. Ms2ger doesn’t have a huge amount of time, so we can spread the reviews around, and maybe I can make a few small improvements. I can volunteer for this and Ms2ger also agreed.

CW: let’s do unanimous consent to add Ryan Hunt as a JS API spec editor.

No objections.
